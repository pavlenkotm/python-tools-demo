# Security Policy

## Supported Versions

Currently supported versions with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please:

1. **Do NOT** open a public issue
2. Email security concerns to: [contact information]
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Security Best Practices

When using this repository's code:

- **Never commit private keys or secrets**
- **Always validate user inputs**
- **Use hardware wallets for production**
- **Audit smart contracts before deployment**
- **Keep dependencies updated**
- **Use testnets for experimentation**

## Smart Contract Security

- Contracts are provided as educational examples
- Audit all contracts before mainnet deployment
- Use established frameworks (OpenZeppelin, etc.)
- Follow security best practices

## Disclosure Policy

- We will acknowledge receipt within 48 hours
- We aim to provide updates within 7 days
- We will credit researchers (if desired)
